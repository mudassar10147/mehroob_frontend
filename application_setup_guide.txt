Option 2: REST API Integration
-------------------------------
Create API client (src/lib/api.ts):
  import axios from 'axios';

  const api = axios.create({
    baseURL: process.env.NEXT_PUBLIC_API_URL,
    headers: {
      'Content-Type': 'application/json',
    },
  });

  // Add interceptors for auth tokens
  api.interceptors.request.use((config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  });

Option 2: Zustand (Simpler, lightweight)
-----------------------------------------
npm install zustand

Example (src/store/cartStore.ts):
  import { create } from 'zustand';

  interface CartState {
    items: CartItem[];
    addItem: (item: CartItem) => void;
    removeItem: (id: string) => void;
  }

  export const useCartStore = create<CartState>((set) => ({
    items: [],
    addItem: (item) => set((state) => ({ 
      items: [...state.items, item] 
    })),
    removeItem: (id) => set((state) => ({ 
      items: state.items.filter(i => i.id !== id) 
    })),
  }));


Option 3: React Context (For simpler state)
--------------------------------------------
Good for auth state, theme, etc.


Recommendation for E-Commerce:
-------------------------------
- Use Zustand or Redux Toolkit for cart management
- Use React Query for server state
- Use Context for theme/auth

NextAuth.js (Recommended)
--------------------------
Installation:
  npm install next-auth

Setup (src/app/api/auth/[...nextauth]/route.ts):
  import NextAuth from 'next-auth';
  import GoogleProvider from 'next-auth/providers/google';
  import CredentialsProvider from 'next-auth/providers/credentials';

  export const authOptions = {
    providers: [
      GoogleProvider({
        clientId: process.env.GOOGLE_CLIENT_ID!,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      }),
      CredentialsProvider({
        // Custom email/password logic
      }),
    ],
    callbacks: {
      async jwt({ token, user }) {
        // Add user ID to token
        return token;
      },
      async session({ session, token }) {
        // Add properties to session
        return session;
      },
    },
  };

  const handler = NextAuth(authOptions);
  export { handler as GET, handler as POST };

10. PAYMENT INTEGRATION
================================================================================

Stripe (Recommended)
--------------------
Installation:
  npm install @stripe/stripe-js stripe

Setup:
1. Create Stripe account
2. Get API keys
3. Set up webhook endpoint

Client-side (src/lib/stripe.ts):
  import { loadStripe } from '@stripe/stripe-js';

  export const stripePromise = loadStripe(
    process.env.NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY!
  );

Server-side API route (src/app/api/checkout/route.ts):
  import Stripe from 'stripe';

  const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

  export async function POST(request: Request) {
    const { items } = await request.json();

    const session = await stripe.checkout.sessions.create({
      payment_method_types: ['card'],
      line_items: items,
      mode: 'payment',
      success_url: `${process.env.NEXT_PUBLIC_APP_URL}/success`,
      cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/cancel`,
    });

    return Response.json({ sessionId: session.id });
  }


Payment Flow:
-------------
1. User adds items to cart
2. User proceeds to checkout
3. Create Stripe checkout session
4. Redirect to Stripe hosted page
5. Handle payment success/failure
6. Create order in database
7. Send confirmation email

19. TROUBLESHOOTING COMMON ISSUES
================================================================================

Issue: Build errors
Solution: Clear .next folder, reinstall dependencies

Issue: TypeScript errors
Solution: Check tsconfig.json, run tsc --noEmit

Issue: Slow development server
Solution: Reduce number of pages, disable sourcemaps

Issue: Image optimization errors
Solution: Check next.config.js image settings

Issue: API routes not working
Solution: Check file naming, verify request methods
